# Current state

This file is the single source of truth for "what's happening right now." Read it first when a session starts — it tells you where the previous session stopped, what to do next, and what's already done so you don't redo it. Update it at the end of every task so the next session can pick up cleanly.

## How to use

1. **At session start:** read this file before anything else. It tells you the active task and the immediate next step.
2. **Before each task:** put the task under "Active task" with enough detail that a cold reader can continue it.
3. **After each task completes:** move it to "Recently completed" (with a test count and a pointer to the history entry), update "Active task" to the next thing, refresh "Next up" if priorities shifted, and update "Test status."
4. **Never use this file for long-term design notes.** Those belong in `Documentation/Plans/`, `Documentation/Architecture/`, or the appropriate algorithm doc. This file is short, operational, and lossy — rewritten often.
