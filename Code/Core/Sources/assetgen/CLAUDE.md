# assetgen — module context

Thin CLI that scans every PAK in a Dune II install and dispatches to `AssetExport`'s per-format extractors. Populates the repo-root `Resources/` tree so the game + docs can reference decoded assets.

## Layout

- `main.swift` — argv parser (`CLI.parse`), install discovery, per-PAK walk, dispatch, top-level logger (`StdoutLogger`).

## Conventions

- No logic beyond argv parsing, PAK iteration, and dispatch. Every format-specific step belongs in `AssetExport.Extractors` so it can be reused / tested without running the CLI.
- `Resources/` is a committed artifact — regenerated by re-running `assetgen`. Do not hand-edit anything it produces.
- Failures on individual PAK entries warn via the logger but do not abort the run — the goal is best-effort full extraction.

## Running

```
cd Code/Core
swift run assetgen                    # extract into repo-root Resources/
swift run assetgen --copy-originals   # also copy raw PAK contents
swift run assetgen -v                 # verbose per-entry log
```

## Key entry points

- `CLI.parse(_:)` — argv → `Options`.
- The `main.swift` tail loop: `archive.entries` → `Extractors.dispatch(entry:context:)`.
