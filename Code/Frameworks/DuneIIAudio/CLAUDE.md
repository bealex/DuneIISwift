# DuneIIAudio

Audio — the `sim → audio` driver. Depends only on `DuneIIContracts` (the `SoundEvent` vocabulary). Never depends on the simulation. `@MainActor` (a UI-side presentation leaf driven from the host's main thread).

The host registers each sound's PCM under a `SoundID` once, then plays `SoundEvent`s; the sink never reads simulation state — it only consumes events the host hands it.

Present (first version):
- **`AudioSink`** protocol — `register(_:sampleRate:pcm8:)` (pre-decode a sound, so a later play has no load latency), `play(_:)`, `stopAll()`. `pcm8` is unsigned 8-bit mono PCM (the VOC sample format).
- **`NullAudio`** — no-op (headless / tests / audio off).
- **`EngineAudioSink`** — the real one (`AVAudioEngine`). **Low-latency + polyphonic**: sounds are pre-decoded into `AVAudioPCMBuffer`s (all resampled to one canonical float32-mono rate, so any buffer plays on any node), and a **pool of player nodes** is kept running, so `play` is a single `scheduleBuffer(at: nil, options: .interrupts)` — it starts at the next render quantum (no perceptible delay) and **mixes** with other voices. Voices are handed out round-robin; only with more than `voices` overlapping sounds does a new one steal the oldest. No audio output device (`start()` throws on a CI box) → it degrades to silent (`play` no-ops). Resampling is manual linear interpolation (adequate for these low-fi samples; avoids `AVAudioConverter`'s `@Sendable` input-block).

**Music** (separate from the SFX sink above). Two interchangeable **backends** behind the `MusicEngine` protocol seam, driven by the unchanged selection policy:
- **`MusicEngine`** / **`MusicBackend`** — the seam: `play(file:song:loop:)`/`stop`/`pause`/`resume` + `onFinished`. `MusicBackend` is `.adlib` (default) or `.midi`; both render the same `(file, song)` index.
- **`ADLMusicPlayer`** (`.adlib`, default) — authentic DOS AdLib FM: the Westwood `DUNE<file>.ADL` files synthesised on an emulated OPL3 chip via the **SwiftOPL3** package (`SwiftOPL3` chip + `WestwoodADL` driver). Producer/consumer: a background thread runs the chip+driver (72 Hz) into a ring buffer; the `AVAudioSourceNode` callback only copies out of the ring (OPL3 DSP is too heavy for the render thread). Cross-thread state lives in `Mutex`es inside a `Sendable` box; the render closure captures no `self` (nonisolated `@Sendable`); no `nonisolated(unsafe)`. FM synth is ~0.67× real-time in debug, ~0.04× in release (after the upstream speedup; release still recommended for margin). See insight `swift-audio-render-thread-mutex`.
- **`MusicPlayer`** (`.midi`) — the mechanism for the MIDI route: the extracted Standard-MIDI songs (`Resources/Audio/Music/DUNE<file>.<song:02d>.mid`) through `AVMIDIPlayer` + a SoundFont/DLS bank (`nil` ⇒ the system's built-in General-MIDI bank). A `generation` token drops stale `AVMIDIPlayer` completion callbacks (it has no cancel).
- **`MusicDirector`** — the policy: a verbatim transcription of OpenDUNE's `g_table_musics` + per-house `musicWin/Lose/Briefing` IDs. `startInGame`/`enterBattle`/`win`/`lose`/`pause`/`resume`, plus the `Music_Play` core, over whichever `MusicEngine` `backend` selects (swappable live — stops, rebuilds, resumes the playing track). Track selection uses an **injected RNG** (host-side analogue of the GUI `Tools_RandomLCG_Range`) — deliberately *not* the sim RNG; music is presentation and outside the determinism contract. See `Documentation/Architecture/Music.md` + `Music.OPL2.md`.

The host (`mapview`/`duneii`) decodes install VOCs (`Voc.decode`), registers them, starts the engine, and plays a feedback sound on each player action; `duneii` additionally owns a `MusicDirector` and drives it from lifecycle events. Tests: `AudioTests` (the 8-bit→float conversion, buffer building + resample, `NullAudio`, graceful no-device) + `MusicDirectorTests` (table + per-house IDs verbatim, filename mapping, seeded random-pool ranges, tracks-exist-on-disk).

The SFX sink does distance attenuation from `SoundEvent.position` (`setListener`); the sim emits `SoundEvent`s for combat/explosions. Phase 7 is complete. Authentic AdLib FM timbre (the OPL2/OPL3 route) is **done** — the `.adlib` backend above (the default), via SwiftOPL3; see `Architecture/Music.OPL2.md`.
