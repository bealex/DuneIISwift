# 03. P2 — Static rendering & scenario model

Status: Draft 2026-04-19 · Follows `02.P1-Complete.md`.

## 1. Goal

Turn the P1 decoders into something runnable. The P2 deliverable has two halves:

1. **Pure-Swift** — a typed, well-tested *semantic* model on top of the byte-level decoders: `Scenario`, `Map`, `HouseLayout`, `UnitSpawn`, `StructureSpawn`. Feeds P3 scenario walking and P4 simulation.
2. **Mac Catalyst + SpriteKit** — an actual runnable app that plays the intro, shows the main menu over `MENTATA.CPS`, and renders a mentat briefing screen using the shipped assets. Uses `AVAudioEngine` + `AVMIDIPlayer` for sound.

The split matters because (1) can be built and CI-tested from the command line while (2) needs Xcode + a running simulator / device.

## 2. Slice 1 — semantic models (pure Swift, testable)

Adds new modules under `Code/Core/Sources/DuneIICore/`:

```
Scenario/
├── Scenario.swift        # top-level: reads an INI and a palette pak
├── HouseLayout.swift     # per-house quota/credits/brain/max_unit
├── UnitSpawn.swift       # parses a UNITS row into a typed struct
├── StructureSpawn.swift  # same for STRUCTURES
└── MapField.swift        # BASIC + MAP sections
Map/
├── Map.swift             # 64×64 tile grid keyed by Formats.IconMap.Group
├── TileResolver.swift    # ICON.MAP + ICON.ICN → render-ready tile lookup
└── LandscapeType.swift   # rock/sand/spice/etc. classification per tile
```

Each new type follows the TDD loop: format doc first, failing test, implement, full suite green, history + insight if non-obvious. For P2 slice 1 the "format doc" is really an algorithm doc under `Documentation/Algorithms/` — these types don't read files directly, they interpret decoded content.

Testing hooks into `SCENA001.INI` (Atreides mission 1) for a real-data smoke test: load → assert `houses["Atreides"].credits == 1000`.

## 3. Slice 2 — rendering (Mac Catalyst, visual)

New Xcode workspace at `Code/DuneIIRemake.xcworkspace` pulling in the SwiftPM package. New app target `DuneIIRemake` with:

```
Code/App/
├── AppDelegate.swift      # UIApplication bootstrap
├── SceneDelegate.swift    # single UIWindowScene
└── GameViewController.swift  # hosts the SKView
Code/Rendering/
├── TileAtlas.swift        # Formats.Icn + IBM.PAL → SKTextureAtlas
├── SpriteAtlas.swift      # Formats.Shp → SKTexture dictionary
├── Scene/
│   ├── Intro.swift        # plays INTRO*.WSA
│   ├── MainMenu.swift     # MENTATA.CPS background + menu buttons
│   └── Mentat.swift       # briefing screen (house-colored)
└── Audio/
    ├── Jukebox.swift      # AVMIDIPlayer queue with SoundFont
    └── Voice.swift        # AVAudioEngine + AVAudioPlayerNode for VOC
```

Milestone: launching `DuneIIRemake.app` plays the intro, advances to the main menu, and "Start a new game" shows a mentat briefing. No gameplay yet — input beyond "click anywhere to continue" is P3.

## 4. Risks / open questions

- **SoundFont licensing** — we need a redistributable GM-compatible SoundFont. GeneralUser GS is CC-licensed and fits; confirm before committing it to `Resources/Audio/SoundFont/`.
- **Mac Catalyst menu bar** — SpriteKit-owning views sometimes need a thin AppKit bridge. Validate by adding File → Open to the main menu.
- **Atlas size** — if `Formats.Icn.TileSet` produces more tiles than a single `SKTextureAtlas` can hold at Retina scale, split by group.
- **ADL decision** — still deferred. If we ship only GM MIDI, we'll document that `.ADL` banks are retained for reference only. Decide here, update the Initial plan accordingly.
