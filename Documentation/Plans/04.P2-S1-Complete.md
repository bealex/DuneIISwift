# 04. P2 slice 1 Complete — Semantic models + simulation foundation

Status: Closed 2026-04-20 · Supersedes the "slice 1" section of `03.P2-Rendering.md`.

## 1. What shipped

The original `03.P2-Rendering.md` §2 scoped slice 1 to seven pure-Swift files: `Scenario`, `HouseLayout`, `UnitSpawn`, `StructureSpawn`, `MapField`, plus `Map` / `TileResolver` / `LandscapeType`. Everything on that list landed, and we kept going past the slice-1 line because the follow-on layers (pools, EMC VM, scheduler, save decoding, `WorldSnapshot`) are also pure Swift and sit on the same CI-testable path.

| Type | Purpose |
|---|---|

### 1.2 Core.Map + TileResolver

| Type | Purpose |

| `Map.Generator.generate(seed:resolver:)` | Bit-for-bit port of OpenDUNE `Map_CreateLandscape` (noise pool → blob spread → stamp → bilinear fill → smoothing → terrain thresholding → spice sprinkling → sprite-index → ICN tile resolution). |
| `TileResolver` | `IconMap`-backed `tileId(_:offset:)` lookup for every `IconMap.Group`. |

## 3. Hand-off to slice 2

Slice 2 is the first time we cross from command-line Swift into an actual runnable app. Scope:

1. A new executable target (`duneii`) that imports `DuneIICore` + `AssetExport` and hosts an `SKView` inside a native window.
2. Three `SKScene` subclasses — `IntroScene` (INTRO\*.WSA playback), `MainMenuScene` (MENTATA.CPS + buttons), `MentatScene` (briefing overlay).
3. `Rendering.TileAtlas` and `Rendering.SpriteAtlas` — pack `Formats.Icn.TileSet` / `Formats.Shp.FrameSet` decoded frames into `SKTexture`s.
4. Audio plumbing: `Audio.Jukebox` (AVMIDIPlayer + SoundFont) and `Audio.Voice` (AVAudioEngine for VOC).
5. An `Installation` discovery path that finds `Repositories/patched_107_unofficial/` (dev build) or reads from a bundled `Resources/Original/` (ship build).

Deviation from the original plan: slice 2 uses **AppKit + SpriteKit** instead of Mac Catalyst UIKit. Rationale: `xcodegen` isn't installed in the dev environment, and hand-authoring `project.pbxproj` for a Catalyst app is more fragile than extending the SwiftPM package with an AppKit executable. The SpriteKit rendering layer — which is the bulk of slice 2 — is identical under either host. Migration path to Catalyst is a project-regeneration step, not a logic rewrite. See `05.P2-S2-P3.md` for the revised plan.

## 4. Success criteria — met

- ✓ `Core.Map` is a 64×64 grid with spice bloom / fog-of-war bitmask per house.

- ✓ EMC VM matches `Script_Run` byte-for-byte.
- ✓ Every vanilla-1.07 `_SAVE00?.DAT` chunk has a typed Swift round-trip decoder (save-write lands in P6).
- ✓ `WorldSnapshot` bridges save → live pools in a single `init(loading:resolver:)` call.
