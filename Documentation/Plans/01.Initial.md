# 01. Initial Plan — Dune II Remake (macOS 26)

Status: Draft · Created 2026-04-19

## 1. Goal

Produce a pixel-faithful, logic-exact recreation of Westwood Studios' *Dune II: The Building of a Dynasty* (v1.07) on **macOS 26**, targeting Apple Silicon. The original gameplay — maps, units, structures, AI, scripts, economy, save format — is preserved exactly; only the presentation layer and a small, clearly-scoped set of post-port improvements are modernized.

### 1.1 Success criteria

1. Every original scenario (`SCEN?00?.INI`) loads and plays identically to Dune II 1.07 including AI build orders, spice economy, and victory triggers.
2. Save files written by this project are byte-compatible with 1.07 save slots (`_SAVE00?.DAT`) — we can load an original save and vice versa.
3. All graphics, sounds, and music are sourced from the original installation (`Repositories/patched_107_unofficial`). This is a private personal project — licensing is not a concern, so we can bundle derived assets (decoded PNG/CAF/SMF, SoundFonts) in `Resources/` freely, or ship the original PAKs directly if convenient.
4. Runs on macOS 26 only, built with Swift 6.3.1, UIKit (Mac Catalyst) + SpriteKit. Native look: proper menu bar, standard dialogs (`UIAlertController` via Catalyst, `NSSavePanel`-style save flows), resizable window.

### 1.2 Non-goals for this phase

- Network / multiplayer play.
- Platforms other than macOS 26.
- Modding tools (PAK editing, scenario editing) — deferred.
- New factions, units, or campaigns.

## 2. Reference material

All non-code references live under `Repositories/`.

| Reference | Purpose |
|-----------|---------|
| `Repositories/OpenDUNE/` | Primary logic reference. C recreation of Dune II; source of truth for scripting (EMC), save format, AI tables, widget layout, format80/format40 codecs. |
| `Repositories/dunepak/` | Rust PAK packer/unpacker. Reference for PAK container format. |
| OpenDUNE wiki `Dune-II-file-formats` | Top-level format index (PAK, SHP, WSA, CPS, PAL, ICN, VOC, XMI/C55/ADL/PCS/TAN, INI, EMC, FNT). |
| `Repositories/patched_107_unofficial/` | User's original 1.07 data. Read-only at runtime; nothing in here is copied into this repo. |

OpenDUNE `INTERNALS.txt` already documents the full 256-color IBM palette organization — this will be re-documented in `Documentation/Formats/Palette.md` in our own words with citations.

## 3. Technology

- **Language**: Swift 6.3.1, strict concurrency, no `@unchecked Sendable`, no `nonisolated(unsafe)`. Follow the user's global code style (`~/Programming/_Scripts/Instructions/CLAUDE.CodeStyle.md`).
- **UI**: UIKit via Mac Catalyst for menus, dialogs, preferences, file pickers, window chrome. All game rendering happens inside a `SKView` embedded in a `UIViewController`.
- **Rendering**: SpriteKit. One `SKScene` per top-level screen (intro, mentat briefing, main menu, gameplay, game-over). Tiles are `SKSpriteNode`s drawn from a single shared `SKTextureAtlas` re-built from extracted ICN tiles at first launch.
- **Audio**: AVFoundation. VOC decoded to `AVAudioPCMBuffer` once at load time; XMI/ADL/C55 converted to Standard MIDI (SMF) and played via `AVMIDIPlayer` using a bundled SoundFont. Any GM-compatible SoundFont we like (e.g. GeneralUser GS, FluidR3, Roland SC-55 emulations) can be bundled directly in `Resources/Audio/SoundFont/`.
- **Concurrency**: game simulation on a dedicated actor (`GameActor`), rendering on `@MainActor`, asset decoding on a background task group. No locks other than `Mutex` from `Synchronization` where strictly needed.
- **Persistence**: two layers — (a) **classic saves** written in the exact Dune II 1.07 `.DAT` layout (backwards compatibility), (b) an optional **native save** wrapper that embeds the classic blob plus modern metadata (thumbnail, timestamp, version).
- **Minimum OS**: macOS 26.0. Swift availability attributes pinned to `@available(macCatalyst 26.0, *)`.

## 5. File format coverage

Every format below gets its own document in `Documentation/Formats/` with (a) layout in our own words, (b) a worked decode example from a real PAK entry, (c) a pointer to the OpenDUNE source that implements it, and (d) the Swift type/name that will decode it.

|--------|---------|--------------------|-------------------------|

| WSA | Pre-rendered animation | `src/wsa.c` | `Formats.Wsa.Animation` |
| CPS | Full-screen compressed image | `src/sprites.c` | `Formats.Cps.Image` |

## 6. Simulation parity

The following sub-systems must match OpenDUNE behavior bit-for-bit. Each gets its own algorithm document.

1. **Map / tile system** — `src/map.c`, `src/tile.c`. 64×64 tile grid, fog of war per house, spice bloom/sandworm logic, landscape type tables (`src/table/landscapeinfo.c`).
2. **Unit pool** — `src/pool/unit.c`, `src/unit.c`. Fixed 102-slot pool. Movement tables (`movementtype.c`), action tables (`actioninfo.c`), unit stats (`unitinfo.c`).
3. **Structure pool** — `src/pool/structure.c`, `src/structure.c`. Placement rules, power grid, credit flow, concrete slab hp, upgrades.
4. **House pool** — `src/pool/house.c`, `src/house.c`. Credit ticks, harvester AI per house, starport delivery.
5. **Team AI** — `src/team.c`, `src/table/teamaction.c`. AI picks actions from a weighted table per scenario phase.
6. **EMC script VM** — `src/script/script.c` and `structure.c`/`unit.c`/ `team.c`. A stack-based VM with ~50 opcodes; each structure and unit has an EMC script that advances one tick per frame.
7. **Scenario loader** — `src/scenario.c`. INI-driven. Spawns map, units, structures, houses from `SCENA00?.INI`/`CHOAM.INI`/`REGION??.INI`.
8. **Save/load** — `src/save.c`, `src/load.c`, `src/saveload/*`. Chunked IFF-like format. Each chunk has a four-char tag and size. Must round-trip a 1.07 save.

## 7. Presentation modernization (scoped)

The only things we change on the presentation side in this phase:

- Native resizable window, aspect-preserved letterboxed viewport.
- Replace in-game popup dialogs (options, save/load, game-over) with UIKit sheets. The Mentat briefing scene, the map screen, the build panel, and all HUD elements keep the original SpriteKit rendering.
- Native file picker for save slot management (rather than the original fixed-slot DAT list).
- Scroll-wheel / trackpad edge-scroll instead of only mouse-at-edge.
- Retina: integer scaling, nearest-neighbor sampling.

Anything else — balance, pacing, animation timing, sound mixing — matches the original.

## 9. Testing strategy

- **Format round trips** — every decoder has an XCTest that decodes a known PAK entry, re-encodes (where applicable), and asserts equality.
- **Golden simulation** — scripted inputs run against the simulation actor for N ticks; resulting state is diffed against a checked-in snapshot. Snapshots are produced once by running OpenDUNE under the same inputs (via a small harness), then frozen.
- **Save parity** — load each of the user's 7 `.DAT` saves, re-serialize, assert byte-identical output.
- **Rendering** — manual only in P2; snapshot tests on the atlas generation step so tile output is deterministic.

## 10. Open questions / risks

Licensing is explicitly out of scope — this is a private personal project, so we can freely bundle any SoundFont, reuse OpenDUNE tables verbatim, and commit original/derived assets. Remaining technical risks:

1. **Mac Catalyst window chrome vs. SpriteKit full-window rendering** may need a thin AppKit bridge for the menu bar. Decide in P2.
2. **EMC opcode coverage** — OpenDUNE has the full opcode table; we can port it directly. Budget: ~1 week in P4 for porting + tests.
3. **Sandworm / spice-bloom RNG** — OpenDUNE uses a specific PRNG (`Tools_Random_?`). We must use the same sequence to match AI behavior bit-for-bit. Document in `Documentation/Algorithms/RNG.md`.
4. **Save format undocumented fields** — a handful of save chunks contain fields OpenDUNE treats as opaque. We preserve them verbatim in the `Save` decoder to keep round-trip parity.
5. **Audio engine choice for Adlib (.ADL)** — XMI converts cleanly to SMF, but ADL is Adlib-native. Options: (a) synthesize via a bundled OPL emulator (e.g. port of DOSBox's dbopl), (b) skip ADL and prefer XMI/C55 banks. Decide in P2.
