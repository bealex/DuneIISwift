## 2. Deviation from the original plan

| Aspect | Plan (01.Initial / 03.P2-Rendering) | Shipped | Why |
|---|---|---|---|
| Host platform | Mac Catalyst (UIKit) | AppKit | `xcodegen` is not installed in the dev environment; hand-authoring `project.pbxproj` for Catalyst was more fragile than extending the SwiftPM package with an AppKit executable. SpriteKit rendering is identical under either host. |
| Intro WSA playback | In slice 2 | Deferred | WSA frame timing + palette cycling need an explicit frame-pump on the scene. Lands with slice-2.1 polish. |
| Audio (Jukebox / Voice) | In slice 2 | Deferred | `AVMIDIPlayer` + SoundFont pick is non-trivial on its own. Lands with slice-2.1 polish. |
| Mentat briefing widget | In slice 2 | Stub (tinted backdrop + caption) | The briefing text comes from `MESSAGE.ENG` + per-house `BRIEF*.WSA` overlay. Lands with P5 (full campaign flow). |
| Scenario unit / structure sprites | In P3 | Deferred | Needs SHP â†’ texture per unit type + per-house palette remap. Lands with P4 simulation when we have live units. |

**Migration path back to Catalyst**: Keep `DuneIIRendering` (it's pure SpriteKit + Core / AssetExport; no AppKit). Rebuild the executable as a Catalyst app in Xcode, replace `GameWindow.swift` with a `UIWindowScene` delegate, and the three scene files work unchanged.

## 3. How to run

```
cd Code/Core
swift build                # 30 s clean; < 5 s incremental
swift run duneii           # opens a window, shows MENTATA.CPS
```

The executable discovers the install by walking up from the current working directory to find `Repositories/patched_107_unofficial/`. Run `duneii` from inside the project tree or it'll exit with a "can't find install" alert.
