See `Documentation/Architecture/Overview.md` §3 for the end-to-end data flow.

**Test coverage**: 90 tests across 15 suites, all green. Every `DecodeError` case has a test, every format variant (modern vs. legacy, compressed vs. raw, continuation vs. regular) is exercised in a synthetic test, and real-data smoke tests short-circuit cleanly when the install isn't present. See `Documentation/Architecture/Testing.md` for the strategy.

**Documentation**: 12 format docs, 11 insights, a monthly history log, a `Documentation/README.md` index, and a rewritten `CLAUDE.md` that mandates a TDD loop for every future feature.

## 3. Lessons learned (→ Insights)

Every non-obvious fact discovered during P1 is captured in `Documentation/Insights/`. The index:

- Codec: `codec-format80-overlapping-copies.md`
- Formats: `format-pak-filename-ascii-8.3.md`, `format-palette-vga-6bit-scaling.md`, `format-shp-offset-table-overlap.md`, `format-shp-row-rle-transparency.md`, `format-wsa-continuation-files.md`, `format-icn-subpalette-indirection.md`, `format-cps-decoded-size-unused-when-compressed.md`, `format-iconmap-double-indirection.md`, `format-ini-case-insensitive-keys.md`, `format-emc-variable-instruction-width.md`, `format-xmi-delay-and-duration.md`
- Audio: `audio-voc-sample-rate-formula.md`
- Workflow: `workflow-tdd-loop.md`

Three generalisations worth repeating:

1. **The reference C source is the spec, not the documentation.** Every "I think this is how X works" that wasn't cross-checked against OpenDUNE turned out wrong at least once.
2. **Forward overlap in LZ-style codecs is not corruption.** It's a run-length encoder. Don't add guards that reject it.
3. **Container formats lie about their sizes.** SHP's offset table, CPS's `decodedSize`, XMI's catalog slot count — every one of them has an off-by-one or a "field is only valid sometimes" quirk that only real data reveals.
