# History — 2026-06-05

- Popovers (Mentat/Options/Scenario/Requirements) now fit small landscape screens: swap fixed heights for maxHeight so the popover clamps to the screen and its List/Form/ScrollView scrolls the overflow, widen contents for landscape, and pin iOS presentation to an anchored popover; new gamePopover(width:maxHeight:) helper in DuneIIClient/PopoverLayout.swift. Presentation-only seam (no oracle/render golden) — neutrality is the 615-test suite staying green; manual checklist in Apps/duneii-ios/README.md.
- Fix Options popover not appearing on iOS: it used arrowEdge .top but its button is at the sidebar's bottom edge, so the anchored popover was placed off-screen below the button — switched to arrowEdge .leading (opens into the map area, matching the Mentat button).
- Options is now full-screen on iOS: present OptionsPopover via fullScreenCover wrapped in a NavigationStack (Options title + Done button to dismiss) instead of a cramped anchored popover; macOS keeps the .leading popover. OptionsPopover gained an isPresented binding for the Done button.
- Options screen is now a single scrolling Form: extracted DebugPanel's controls into a wrapper-less DebugToggleRows view so the Options Form holds scenario/speed + debug toggles (+ macOS Settings link) as sections of one Form, instead of two stacked Forms each scrolling independently.
- iOS Options scenario picker: the Scenario row is now a single NavigationLink that pushes ScenarioPicker full-screen onto the Options NavigationStack (was a fixed-300pt nested popover that hid most content and a row that stretched vertically); ScenarioPicker dismisses via @Environment(.dismiss) and fills width on iOS, macOS keeps the labelled-button popover.
- Every action surfaced for a selected item now carries a shortcut badge: ActionType.shortcut maps .stop→S (Stop appears in harvester/MCV/carryall menus; the 's' key already stopped them) and the Palace super-weapon button gained an 'L' (Launch) badge + a GameScene 'l' keyDown case (launchSuperWeapon is self-guarded). Only the AI-only actions (ambush/hunt/die — never in actionsPlayer) stay shortcut-less.
- Scenario picker now uses a standard List: the house segmented picker is the top section and each campaign level is its own Section of mission rows, replacing the hand-rolled ScrollView/VStack/Button layout (List handles scrolling, so the macOS inner maxHeight-360 frame is gone).
- iPhone sidebar now scrolls as a whole: extracted GameSidebar.sidebarBody with a platform split — iOS wraps header/minimap/selection/build/bottomBar in one ScrollView (landscape sidebar can exceed screen height), macOS keeps the fixed layout with only the build list scrolling and the button bar pinned.
- iPhone input + Options polish: (1) pin the sidebar button row below the scroll (Options/Mentat/Save/Load always reachable); (2) 1:1 finger-following pan — drag uses view-space deltas through scroll() instead of double-dividing scene deltas by zoom; (3) continuous two-finger pinch zoom via new GameModel.setZoom, capped at Viewport.maxZoom now 8x (was 16x); (4) a Zoom slider (1–8x) under the speed picker in Options; (5) new placement gestures — dragging the footprint follows the finger, a tap on the footprint commits the build (PlacementState.contains), a tap outside cancels, and the placement long-press cancel is gone.
- Fix pinch-zoom onset on iPhone: gate single-finger pan behind a 0.09s settle window (GameScene.panEnabled) so the first finger of a pinch can't lurch the map before the second finger lands; a second finger arriving in the window cancels the pending pan, and pre-settle finger drift is discarded (touchLast keeps tracking) so panning starts jump-free.
- Fix pinch-zoom jiggle + bump pan settle delay: pinchDistance now measures finger separation in view (screen) points instead of scene points — scene coords scale with the camera, so the changing zoom fed back into the distance and the magnification oscillated; bumped panSettleDelay 0.09s→0.18s so the second pinch finger is reliably caught before a one-finger pan starts.
