# The player's out-of-money construction does NOT auto-resume — only the repair pad does

**Finding:** when a build runs out of money, OpenDUNE/1.07 sets `onHold` **only for the player** (`structure.c:266` gates on `g_playerHouseID`) and never clears it automatically. The AI never gets `onHold` set, so its build simply idles and resumes on its own when money returns. The *only* auto-resume in `Structure_Tick` is for the **repair pad** (`structure.c:302-305`, `else if (h->credits != 0) onHold = false`). The player resumes a held construction-yard/factory build by clicking the item — the GUI `STR_ON_HOLD` case (`gui/widget_click.c:107-111`) clears `repairing`/`onHold`/`upgrading`; the inverse `STR_D_DONE` click (`widget_click.c:124`) sets `onHold` to pause.

**Why it matters:** it reads like a bug ("my build halted and never restarted when I got money"), so it's tempting to add an auto-resume to the player factory branch — that would diverge from the oracle and break parity. The faithful UX is a manual Pause/Resume toggle. Calling `Structure_BuildObject` again does **not** resume: it returns early on an already-linked build (`structure.c:175` / our `UnitCombat.structureBuildObject` `linkedID != 0xFF` guard) without touching the flag.

**Evidence:** OpenDUNE `src/structure.c:264-306`, `src/gui/widget_click.c:107-126`, `src/gui/gui.c:744-752`. Our faithful transcription: `GameState.structurePauseBuild`/`structureResumeBuild` (`GameState+Lifecycle.swift`), the `tickStructure` factory branch + repair-pad auto-resume (`GameState+Lifecycle.swift:625/693`), `Command.pauseBuild`/`.resumeBuild`, `duneii` `Panels.buildSection` Pause/Resume buttons. Tests: `StructureProductionTests.pauseThenResume` + `.resumeStillBrokeReholds`.

**How to apply:** never auto-resume a player build when credits return — that's a deliberate divergence, confirm with the user first. Model it as click-to-toggle (`STR_D_DONE` pause / `STR_ON_HOLD` resume), and remember a resumed build re-holds the same tick if still underfunded. [[build-getbuildable-upgradelevel-gate]]
