# The oracle scenario harness ticks only the four GameLoop phases — not Explosion/Animation

**Finding:** OpenDUNE's scenario parity harness (`src/parity.c` `Parity_RunScenario`) runs **only** `GameLoop_Team/Unit/Structure/House` per tick — it never calls `Explosion_Tick` (and would not call any other visual subsystem the full `GameLoop_Main` ticks). The full game ticks them; the parity scenario runner does not.

**Why it matters:** any subsystem we tick on our side that **draws RNG** (the shared `Random_256` / `RandomLCG`) but the oracle harness doesn't will desync every golden that triggers it. `Explosion_Tick` draws `Tools_RandomLCG_Range` (`SET_RANDOM_TIMEOUT`) and `Tools_Random_256` (`TILE_DAMAGE`, `SET_ANIMATION`); `attack-structure` destroys a windtrap → an `EXPLOSION_STRUCTURE` → an LCG draw. Ticking explosions unconditionally in `Simulation.tick()` would have broken that 400-tick golden. (Note the *start* call is safe: `Explosion_Start` / `Animation_Start` draw no RNG and the oracle does call them, so populating the pools is golden-neutral — only the *tick* diverges. Our already-shipped `animationTick` survives only because no golden scenario reaches an animation command that draws RNG within its compared window.)

**Evidence:** `Repositories/OpenDUNE/src/parity.c:436-439,730-733` (phase list, no `Explosion_Tick`); `Repositories/OpenDUNE/src/explosion.c:233,92,183` (the RNG draws); our gate `Code/Frameworks/DuneIISimulation/Simulation.swift` (`tickExplosions`, default `false`) + `Code/Tests/ScenariosTests/ScenarioExplosionTests.swift` (animates when ticked; sprite stays 0 when gated off). See `Documentation/Algorithms/Explosion.md`.

**How to apply:** when porting a visual/secondary subsystem that draws RNG (explosions, map blooms, sandworm shimmer, …), gate its per-tick advance behind a flag that is **off** on the golden/oracle-matched path and **on** only in the lab — or, for full parity, first teach `parity.c` to tick it too and regenerate the goldens. Wiring the RNG-free *start* into the deterministic path is fine; ticking is the hazard.
