# Foot-unit corpses are an overlay animation (a separate table from structure animations)

**Finding:** a dying foot unit's corpse is an **overlay animation** started by `Script_Unit_StartAnimation` (op 0x04) just before `Unit_Die` removes the body. It runs from `g_table_animation_unitScript1` (3-frame infantry: soldier/trooper) or `g_table_animation_unitScript2` (other foot units) — *different* command tables from `g_table_animation_structure` (the building ground-cycle). The commands are `SET_OVERLAY_TILE → PAUSE 600 → SET_OVERLAY_TILE → PAUSE 600 → STOP`, so the corpse rides the map tile's **`overlayTileID`** (icon group 4) and lingers ~1200 ticks, then `STOP` clears the overlay (`Animation_Func_Stop` zeroes `overlayTileID` on revealed tiles). Row 0/1 = on sand, 2/3 = on rock (`variables[1] == 1` adds 2).

**Why it matters:** our `Animation` pool only knew the `structure` table and `startAnimation`/`SET_OVERLAY_TILE` were stubs, so a killed trooper vanished instantly — no corpse. Supporting it needed (a) porting the two `unitScript` tables, (b) a `kind: AnimationKind` selector on `Animation` so `animationTick` reads the right table, (c) implementing `SET_OVERLAY_TILE` (set/clear `overlayTileID`, gated on `isUnveiled`), and (d) wiring op 0x04 to start it. The corpse renders for free through the existing overlay-composite path (`FrameInfo.overlaySpriteIndex` → `FrameComposer.cell`).

**Golden-safe:** `Animation_Start` is RNG-free, so starting the corpse in the golden path (op 0x04 runs in `GameLoop_Unit`) draws nothing; only `animationTick` (gated by `tickAnimations`, off in the oracle harness) advances it and draws the `PAUSE` jitter. All 483 tests stayed green.

**Evidence:** `Frameworks/DuneIIWorld/Model/Animation.swift` (`AnimationKind`, `unitScript1/2`); `GameState+Animation.swift` (table switch, `animationSetOverlayTile`, the overlay clear in `animationStop`); `UnitScriptFunctions.startAnimation`. Test: `InteractionTests.soldierLeavesCorpse`. OpenDUNE: `script/unit.c` `Script_Unit_StartAnimation`, `table/animation.c:66/93`, `animation.c` `Animation_Func_SetOverlayTile`/`_Stop`.

**How to apply:** the corpse overlay tile id is truncated into `overlayTileID` (UInt8/7-bit) — fine for the small icon-group-4 ids, but the same caveat as the fog-veil overlay ([[world-fog-veil-overlay-off-by-one]]). When adding any new overlay-driven animation, give it an `AnimationKind` + table and implement the overlay command; don't reuse the structure table.
