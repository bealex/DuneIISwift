# Structure_GetBuildable gates on the factory's upgradeLevel — set at Structure_Create

**Finding:** whether a factory can build an item is gated (`structure.c:1834` `Structure_GetBuildable`) by `s->upgradeLevel >= item.upgradeLevelRequired`, **not** just by house/prereqs. The factory's starting `upgradeLevel` is assigned in `Structure_Create`: a **Harkonnen light factory starts at `upgradeLevel 1`** (`UnitCombat.structureCreate`, mirroring `structure.c`), and the **quad** (the Harkonnen light-vehicle unit) has `upgradeLevelRequired: 1`. So a Harkonnen light factory built through the real path can make the quad immediately; a *synthetically* allocated one left at `upgradeLevel 0` lists **nothing** (the trike is Atreides/Ordos-only via `availableHouse`, and the quad fails the upgrade gate). An item that fails only the upgrade gate is "upgrade-first" (`available == -1`) — our `buildables` omits those.

**Why it matters:** a test (or any code) that hand-builds a factory with `structureAllocate` + a few fields, instead of `structureCreate`, gets `upgradeLevel 0` and an empty/short buildable list — easy to misread as a `GetBuildable` bug. The per-house starting upgrade level (and the AI's "max it immediately" loop) is real game state that the buildable filter depends on.

**Evidence:** `Frameworks/DuneIISimulation/StructureBuild.swift` `Simulation.buildables(forStructure:)` (the port); the Harkonnen light-factory `upgradeLevel = 1` special in `UnitCombat.structureCreate`; `UnitInfo[.quad].o.upgradeLevelRequired == 1`, `availableHouse == 63`. Test `StructureBuildTests.buildablesUnits` sets `upgradeLevel = 1` to match the real factory. Related: [[render-fog-two-passes]] (unrelated) — see `Documentation/Architecture/BuildGUI.md`.

**How to apply:** when exercising the build path in a test, set the factory's `upgradeLevel` to what `Structure_Create` would (Harkonnen light factory → 1; AIs → maxed), or build it via `structureCreate`. When debugging an empty build menu in the app, check the selected factory's `upgradeLevel` before suspecting the filter.
