# A unit's death animation + sound come from THREE distinct sources — don't add a "default death explosion"

**Finding:** there is no single "unit death" effect. A destroyed unit's audiovisual is produced by three independent mechanisms, so it legitimately differs (and is sometimes silent) by unit type and death cause:
1. **Death explosion sprite + boom** — the unit's *DIE EMC script* runs `Script_Unit_ExplosionSingle` (op 0x0E, `UnitImpact.explosionSingle`) → `Map_MakeExplosion(explosionType, hitpoints=unit's full HP)`. The explosion *table* may carry a `playVoice` command (the boom). Vehicles (tankExplode 6 / wheeledVehicle 10 / tankFlames 9) all `playVoice`; this is the audible vehicle death. `Script_Unit_Die` (0x0F, `UnitImpact.die`) adds **only** the saboteur's 300-dmg blast — NOT a generic explosion.
2. **Foot-unit corpse** — infantry/troopers die to a silent *corpse overlay* animation (`unitScript1/2` tables, insight `sim-unit-corpse-overlay-animation`), not an explosion ⇒ **no boom**.
3. **Spoken "unit destroyed" announcement** — OpenDUNE's `Unit_Damage` lethal branch calls `Sound_Output_Feedback` (saboteur→20, else `houseID+14` for the player/late campaign else 13), gated on `!ui->flags.noMessageOnDeath && wasAlive`. `noMessageOnDeath` is set on projectiles/sandworm/frigate (intentionally silent). **This is still a SEAM in our engine** (`UnitImpact.damage`, the `// SEAM: Sound_Output_Feedback death cue` line) — the announcement never plays.
4. **(Edge) worm-swallowed units** are `Unit_Remove`d directly with no DIE action, so none of the above fire — the only effect is the worm's own `EXPLOSION_SANDWORM_SWALLOW` gulp (was itself a seam; fixed 2026-06-01 in `UnitCombat.fire`).

**Why it matters:** "destruction is sometimes silent / has no animation" is expected, not one bug. A tempting "fix" — make `die()` spawn a default explosion for every unit — is WRONG: the DIE script already runs `ExplosionSingle`, so it double-explodes (and foot units would wrongly explode instead of leaving a corpse). The real gaps are narrow: the swallow seam (fixed) and the spoken death-announcement seam (#3, still open — a multi-word `Sound_Output_Feedback` voice sequence, a separate follow-up).

**Evidence:** `UnitImpact.swift` `damage()` (lethal branch + the SEAM comment), `explosionSingle()`/`die()` (only saboteur blasts); `Model/Explosion.swift` `ExplosionTables` (which types `playVoice`); `UnitCombat.swift` sandworm `fire` case; OpenDUNE `src/unit.c:1530` `Unit_Damage` (the `Sound_Output_Feedback` death cue + `noMessageOnDeath`/`alive` gates), `src/table/UnitInfo.c` (`noMessageOnDeath`). Tests: `SandwormNativesTests.eatStartsSwallowExplosion`.

**How to apply:** to implement the missing spoken death announcement, emit through the `pendingFeedback` queue (as `Structure_HouseUnderAttack` does for id 48) from the `Unit_Damage` lethal branch with the OpenDUNE index selection, and map those feedback indices to voices in the host — do NOT add an explosion in `die()`. Keep it golden-neutral: `pendingFeedback.append` draws no RNG.
