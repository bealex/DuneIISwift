# Cross-engine goldens for GUI/placement actions: headless gotchas + the in-transport dump skip

**Finding:** A scenario golden can exercise a *GUI* action (build + place a structure) by adding a command to both engines — but the oracle harness is headless, so any player-house GUI text path crashes, and the per-tick dump silently drops in-transport units.

**Why it matters:** Three traps cost real time porting the `refinery-harvester` golden:
1. **The full `Structure_BuildObject` crashes for the player house** — its "production started" / underpowered `GUI_DisplayText(String_Get_ByIndex(...))` dereferences strings the parity harness never loads. The oracle `Scen_BuildPlace` must replicate only the *headless state-setup* (`Structure_Create` off-map → `Structure_Place`), exactly as the Swift `structureBuildObject` treats that text as a seam. And `House_CalculatePowerAndCredit` *also* has a player-house underpowered-warning `GUI_DisplayText` — so the scenario must keep the player **powered** (add a windtrap) or `Structure_Place`'s power recompute crashes.
2. **`Unit_Find` skips `isNotOnMap` units when `g_validateStrictIfZero == 0`** (`pool/unit.c:42`) — and the dump runs at strict==0. So a freshly-spawned, carryall-ferried harvester (created off-map) appears in *neither* engine's dump. Match the Swift snapshot to it (filter out `isNotOnMap`); prove the spawn happened via the carryall + the house `unitCount` instead.
3. **`Unit_CreateWrapper` returns the cargo in OpenDUNE but the carryall in our port.** OpenDUNE returns `unit` (the harvester); `UnitCombat.unitCreateWrapper` returns `carryall`. So `viewport.c:210`'s `u->originEncoded = encode(refinery)` sets the **harvester's** origin in OpenDUNE but our `structurePlaceReady` sets the **carryall's** — a latent bug (the delivered harvester's home refinery is left unset). Invisible to this golden (the harvester is in-transport-skipped and `originEncoded` isn't dumped). **TODO:** make `unitCreateWrapper` return the cargo + re-point the `originEncoded` callers, then re-run the goldens.

**Evidence:** oracle `Scen_BuildPlace` + `--parity-place` (`parity.c`); Swift `ScenarioGoldenTests` `place:` specs + the `isNotOnMap` snapshot filter; `refinery-harvester.ini`; OpenDUNE `structure.c` (`Structure_BuildObject`/`House_CalculatePowerAndCredit`), `unit.c:1761` (`Unit_CreateWrapper`), `pool/unit.c:42` (`Unit_Find`).

**How to apply:** To golden a placement/GUI action, add a symmetric command to `parity.c` + the Swift runner that does only the deterministic state-setup (no GUI text), keep the player powered, and compare the *observable downstream* state (placed structures, spawned on-map units, `unitCount`) — not the transient in-transport units.
