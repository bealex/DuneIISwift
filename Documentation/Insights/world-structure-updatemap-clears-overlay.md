# Structure_UpdateMap clears the tile overlay — a building covers any crater/wall under it

**Finding:** `Structure_UpdateMap` doesn't just write each footprint tile's `groundTileID`; it also clears a *real* overlay sitting on that tile — `if (Tile_IsUnveiled(t->overlayTileID)) t->overlayTileID = 0` (`structure.c`). `Tile_IsUnveiled(id)` is true outside the 16-tile fog-of-war run `[g_veiledTileID − 15, g_veiledTileID]`, so this removes a missile crater / wall overlay (a building covers it) while leaving a fog-veil overlay intact.

**Why it matters:** the renderer composites the tile **overlay over the ground** (`FrameComposer.cell`), and a structure is stamped into `groundTileID`. If `structureUpdateMap` only writes the ground tile, a crater overlay left on the tile (from an earlier impact, before the building was placed there) draws *on top of* the structure — the visible bug was a missile crater rendered over a gun turret. A crater never forms *on* a structure tile (`drainCraters` skips `craterType == 0` landscapes), so the only way a structure tile carries a crater is a building placed on a previously-cratered tile — exactly the case this clear handles.

**Why it's golden-safe to add:** OpenDUNE already clears it, so the cross-engine goldens were generated against the cleared behaviour; ours passed without the clear only because no golden scenario builds on a crater within its window (the dumps don't include map overlays anyway). Adding the clear makes us *more* faithful and keeps every golden byte-identical.

**Evidence:** fix `DuneIIWorld/State/GameState+Animation.swift` (`structureUpdateMap`, the `overlay != 0 && (overlay > tileIDs.veiled || overlay < tileIDs.veiled − 15)` clear); test `WorldTests/StructureUpdateMapTests` (crater cleared, fog veil preserved); OpenDUNE `structure.c` `Structure_UpdateMap` + `sprites.c:477` `Tile_IsUnveiled`. The same `Tile_IsUnveiled` overlay-range test already lives in `GameState+Explosion`/`GameState+WallSlab`.

**How to apply:** the tile `overlayTileID` slot is multiplexed — fog veil vs. real overlays (crater/wall/concrete). Any code that writes a tile's ground under a covering object should clear a *real* overlay there (guard with the `Tile_IsUnveiled` range), and any code that distinguishes fog from overlays must use that range test, never a bare `overlayTileID != 0`.
