# Fog masking is two independent passes: terrain black-fill AND per-entity skip

**Finding:** in `viewport.c`, fog of war hides things in **two separate** places, and both are required. (1) The **terrain pass** fills a fully-veiled cell with a black rectangle: `if (g_veiledTileID == t->overlayTileID) { GUI_DrawFilledRectangle(..., 12); continue; }` (`viewport.c:389`). (2) **Every entity pass independently re-checks the tile and skips** — units (`viewport.c:421`/`484`), explosions + smoke (`644`/`681`), the sandworm pass — each does `if (!g_map[curPos].isUnveiled && !g_debugScenario) continue;`. The black terrain rectangle does **not** cover units, because units are drawn in *later* passes on top of the terrain; without the per-entity `isUnveiled` skip, every enemy in the fog draws over the black veil.

**Why it matters:** our renderer had only pass (1) — `FrameComposer.cell` blacked out veiled terrain, so "detected tiles seemed OK" — but `FrameComposer.sprites`/`updateBlurs`/the health-bar overlay drew **all** entities regardless of fog. With `showFog` on, the whole enemy roster floated on the black map ("masking is totally wrong"). It's easy to miss because the terrain half looks right and the entity half is a *different code path*.

**Evidence:** `Code/Frameworks/DuneIIRenderer/FrameComposer.swift` — `isHiddenByFog(_:worldX:worldY:showFog:)` returns `showFog && !tiles[ty*w+tx].isUnveiled`; `sprites(...)` filters units + effects through it, `SpriteKitRenderer.updateSprites`/`updateBlurs` pass `showFog`, and `duneii`'s `GameScene.updateHealth` skips hidden entities' bars. Structures need no extra skip — they're baked into the ground tiles, so the terrain pass blacks them out. Test: `FrameComposerTests.fogMasksEntities`. Related: [[render-incremental-appearance-key]], [[world-fog-veil-overlay-off-by-one]].

**How to apply:** any new drawable laid **over** the terrain (a new effect, a HUD marker, a projectile) must be fog-masked at its own pass via `isHiddenByFog` — relying on the terrain black-fill to hide it is wrong, since it renders later and on top. Verify on the running app with fog on, not just the terrain golden.
