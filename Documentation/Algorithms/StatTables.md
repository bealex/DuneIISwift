# Static stat tables

Dune II's per-type constants live in **hardcoded C tables** (`src/table/*info.c`), not on disk — so they are ported as Swift `let`s in `DuneIIWorld/Stats/`, one struct + `table` array per OpenDUNE `g_table_*`. `ObjectInfo.available` and a few other "static" fields are actually mutable runtime state and will be hoisted into `GameState` when those tables land; the pure-constant tables below are plain `let`s.

Each table is **golden-verified field-for-field** against the oracle: `opendune --parity-golden=<dir>` dumps one `<name>-golden.jsonl` per table (one JSON object per entry), and the Swift test asserts every field of every entry. A transcription typo fails the test. See `../Architecture/FunctionParityHarness.md`.

## Ported so far

- **`HouseInfo`** (`house.h` / `houseinfo.c`) — keyed by `HouseID` (Contracts). Per-house toughness, degrading chance/amount, minimap colour, special-weapon countdown + type, starport delivery time, filename prefix char, win/lose/briefing music, voice file. Fixture `houseinfo-golden.jsonl`, suite `HouseInfoGoldenTests`.
- **`LandscapeInfo`** (`map.h` / `landscapeinfo.c`) — keyed by `LandscapeType` (15). `movementSpeed[MovementType]` (6), wobble/build-validity/sand/spice flags, crater type, radar colour, sprite id. Fixture `landscapeinfo-golden.jsonl`, suite `StatTableGoldenTests`.
- **`ActionInfo`** (`unit.h` / `actioninfo.c`) — keyed by `ActionType` (14). String id, name, switch type (queue/immediate/subroutine), `SelectionType`, foot-unit sound id. Fixture `actioninfo-golden.jsonl`, suite `StatTableGoldenTests`.

Supporting enums introduced with these: `HouseID` (Contracts); `LandscapeType`, `MovementType`, `ActionType`, `SelectionType` (World).

- **`ObjectInfo`** (`object.h`) — the fields common to a unit and a structure: abbreviated/full string ids, `name`, optional `wsa` filename, a 13-flag bitfield (`hasShadow`/`factory`/`notOnConcrete`/`busyStateIsIncoming`/`blurTile`/`hasTurret`/`conquerable`/`canBePickedUp`/`noMessageOnDeath`/`tabSelectable`/`scriptNoSlowdown`/`targetAir`/`priority`), spawn chance, hitpoints, fog radius, spriteID, build credits/time, `availableCampaign`, `structuresRequired` (a `FLAG_STRUCTURE_*` bitmask, `UInt32`; `FLAG_STRUCTURE_NEVER` = `0xFFFFFFFF`), sort priority, upgrade level, the four `actionsPlayer`, the (currently always-0) `available`, hint string id, build/target priorities, and the `availableHouse` (`FLAG_HOUSE_*`) bitmask. Modelled as `ObjectInfo` with nested `ObjectInfo.Flags`. (`available` is really mutable runtime state; kept as a `let` initial value here until `GameState` lands, then hoisted.)
- **`UnitInfo`** (`unit.h` / `unitinfo.c`, `UNIT_MAX` = 27) — embeds an `ObjectInfo`, plus pool `indexStart`/`indexEnd`, a 13-flag `UnitInfo.Flags` bitfield (`isBullet`/`explodeOnDeath`/`sonicProtection`/`canWobble`/`isTracked`/`isGroundUnit`/`mustStayInMap`/`firesTwice`/`impactOnSand`/`isNotDeviatable`/`hasAnimationSet`/`notAccurate`/`isNormalUnit`), dimension, `movementType` (`MovementType`), animation/moving-speed factors, turning speed, ground/turret sprite ids (turret `0xFFFF` = none), `actionAI` (a raw `UInt16`; can be `ACTION_INVALID` = 0xFF), `displayMode` (`DisplayMode`), destroyed-sprite id, fire delay/distance, damage, `explosionType` (raw `UInt16`), `bulletType` (raw `UInt8`; `UNIT_INVALID` = 0xFF), bullet sound (`0xFFFF` = none). Fixture `unitinfo-golden.jsonl`, suite `UnitStructureInfoGoldenTests`.
- **`StructureInfo`** (`structure.h` / `structureinfo.c`, `STRUCTURE_MAX` = 19) — embeds an `ObjectInfo`, plus `enterFilter` (`FLAG_UNIT_*` bitmask, `UInt32`), credits storage, `powerUsage` (**signed** `Int16` — negative = produces, e.g. windtrap −100), `layout` (`StructureLayout`), `iconGroup` (raw `UInt16`), `animationIndex[3]`, `buildableUnits[8]` (raw `UInt8`, `UNIT_INVALID` sentinels), `upgradeCampaign[3]`. Fixture `structureinfo-golden.jsonl`, same suite.

Seam enums introduced with these: `UnitType` (`UNIT_MAX` = 27) and `StructureType` (`STRUCTURE_MAX` = 19) in **Contracts** (shared sim↔render vocabulary, alongside `HouseID`); `DisplayMode` and `StructureLayout` in **World** (next to `MovementType`).

The two big tables are field-printed by the oracle's `Golden_UnitInfo` / `Golden_StructureInfo` (reading the real compiled `g_table_*`), so the golden fixtures are the authoritative numeric values; the Swift port reproduces them and the suite also spot-checks a few hand-known values (e.g. Carryall HP 100 / 800cr, Devastator HP 400, windtrap `powerUsage` −100) as an independent anchor.

- **`StructureLayoutInfo`** (`structure.h` / `structureinfo.c`, 7 layouts) — the structure layout geometry, consolidating OpenDUNE's six `g_table_structure_layout*` arrays into one struct keyed by `StructureLayout`: `tiles[9]` (packed-tile offset of each tile; `+64` = one map row down), `edgeTiles[8]`, `tileCount`, `tileDiff` (`Tile32` sub-tile extent), `size` (`XYSize` w×h), `tilesAround[16]` (the surrounding ring, signed `Int16`). Drives `Structure_UpdateMap`, build-location validity, wall connection. Fixture `structurelayout-golden.jsonl`, suite `StatTableGoldenTests`. Also adds `XYSize` (World).
- **`ActionInfo.actionsAI`** — `g_table_actionsAI[4]` (`unitinfo.c:12`): the AI default-action rotation (`hunt`, `areaGuard`, `ambush`, `guard_`). Fixture `actionsai-golden.jsonl`.

## Not yet ported (next chunks)

- The `ICM_ICONGROUP_*` and `EXPLOSION_*` enums (currently stored raw on `UnitInfo`/`StructureInfo`), movement-type names, damage/explosion/animation tables, team actions.
- Reconcile `DuneIIRenderer.StructureCatalog` against `StructureLayoutInfo` — the renderer can't import World (it depends only on Contracts + Formats), so its layout mirror stays independent; a cross-target test (RendererTests + World) could assert they agree.
